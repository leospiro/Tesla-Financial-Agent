# Tesla Financial QA System

一个基于大语言模型 (LLM)、混合检索 (Hybrid RAG) 与智能体推理 (Agent) 架构的特斯拉历年财报智能问答系统。针对特斯拉极为庞杂的冗长 10-K（年报）与 10-Q（季报）设计，支持跨期数据对比、跨表数字计算与深层文本逻辑关联查询。

---

## 快速运行指南 (How to Run)

本项目建议在 `backend` 的 Conda 环境下执行，核心三方依赖见根目录 `requirements.txt` 和 `main-requirements.txt`。

### 1. 环境准备
确保激活环境及安装依赖：
```bash
conda activate backend
```

配置必要的大模型运行 KEY，本项目默认集成了兼容于 OpenAI 协议的接口（代码中默认对接并适配阿里云百炼 `DashScope`）：
```bash
# Windows PowerShell
$env:ALI_API_KEY="sk-xxxx"  # 你的阿里云百炼 API KEY

# 如更换底座大模型（推荐自带 Tool Calling 能力的大模型）：
$env:LLM_MODEL_NAME="qwen3.5-plus"
```
（你可以直接使用项目根目录下的 `.env.example` 复制一份为 `.env` 进行管理配置）

### 2. 数据入库与索引建立 (Ingestion)
将所有目标 PDF 文档放入 `myProject/reports/` 目录下，运行建立混合检索库：
```bash
python ingest.py
```
> **提示**：为避免失败带来的时间成本，本项目已实现**断点续传**与**解析结果离线缓存（Pickle）**。如果报错中断，再次执行时将秒速跳过解析阶段，并接着上次的 Batch 进度自动补写向量入库。

### 3. 启动交互式网页服务界面 (User Interface)
我们提供了一个开箱即用、基于 Gradio 构建在本地的 ChatBot：
```bash
python run.py
```
终端启动后，请使用自带浏览器打开本地对应地址（通常为 `http://127.0.0.1:7860/`），即可在极简界面下与特斯拉财务助理直接交互。

### 4. 自动化高阶测评测验 (Evaluation)
内置包含 5 道超级复杂财务跨表综合题的自动压测集，在后台全自动化评测：
```bash
python eval_testset.py
```
运行该命令不会弹出 UI 界面，生成的推理调用流转日志将自动留存。

## 系统核心设计抉择 (System Design Decisions)

这套系统采用了自下而上的架构方案，拒绝做简单的“文档问答”，主要实现了以下三大核心策略：

### 1. 数据处理与解析策略
- **PDF 增强解析（图表转 Markdown）**：
  纯文本提取引擎（如原生 `pypdf`）会在遇到财报中高密度的横表与竖表时发生“行列错位坍缩”，导致大模型直接失去表头的年份和科目的关联（不知道“12,500”是指营业成本还是研发利润）。为了彻底解决，我们启用了 `pdfplumber` 获取几何结构坐标，将普通断落解析为文本的基础上，**将所有报表表格转换为带有 `| - |` 的强对齐 Markdown 结构化内容**，彻底根治丢表错表病根。
- **自然段级语义分块 (Semantic Chunking)**：
  抛弃低效且极易切破数字表的机械定长方案（Fix-Length）。基于财务报表的强格式化特征，构建了**基于文档原有 SEC 报表头标志位**（如正则匹配 `Item 1.`、`PART II`、空白回车行）作为缝隙切刀的独立切分，确保任意一张大表格都将作为一个不被“生硬腰斩”的纯净上下文块单元。
- **强制注入四个维度金矿元数据**：
  针对所有的 Chunk 片段，都在内存中给打上四重印记：`year`, `quarter`, `report_type`, `section`。极大降低了后续的检索底噪。

### 2. 混合检索策略 (Hybrid Retrieval)
- **稀疏向量（BM25） + 密集向量（ChromaDB）结合**：
  采用 Reciprocal Rank Fusion 技术栈：
    - **密集向量**（捕获“描述一下公司应对困难挑战做出的改变？”）；
    - **稀疏关键字 BM25**（对极其生僻的专有名词或特定数字——例如查询 “Free Cash Flow” 或硬搜索 “2022 Q3” 达成物理级别的命中兜底）。

### 3. 生成与问答策略 (Agent Framework)
- **采用具有规划自主权（Planning & Reasoning）的 Agent 执行器调度**：
  传统的“Retriever QA（查表就回答）”框架无法应对像“计算 2022 年四个季度某项费用的总和，并与 2021 全年对比”的问题。
  所以我们将大模型降维成了“大脑”，为它配置了两个工具：
  - `financial_document_search` (混合搜索文献提取子工具)
  - `math_calculator` (算力外挂：计算器)
  系统现在**拥有循环与多次多步执行调用的自制权**：大模型需要规划“它应当先分别找四次 Q1/Q2/Q3/Q4的数据，然后把它传给 `math` 计算器加和，最后在返回结果”，显著解决了数学幻觉与记忆短暂的问题。
